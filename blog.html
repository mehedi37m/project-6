Discuss the scope of var, let, and const

var Scope: Function Scope: Variables declared with var are function-scoped, meaning they are accessible within the function in which they are declared. They are not limited by block boundaries (e.g., if statements or loops) and can be accessed throughout the entire function. Global Scope: If a var variable is declared outside of any function, it becomes a global variable and is accessible throughout the entire script.

let Scope: Block Scope: Variables declared with let are block-scoped, which means they are accessible only within the nearest enclosing block (a pair of curly braces {}), such as an if statement or a loop. They are not accessible outside of that block. Global Scope: let variables declared at the top level (outside of any function) are not added as properties to the global object and are only accessible in the global scope.

const Scope: Block Scope: Variables declared with const are also block-scoped and follow the same block scope rules as let. They are only accessible within the nearest enclosing block and not outside of it. Global Scope: Like let, const variables declared at the top level are not added as properties to the global object and are only accessible in the global scope.

Tell us the use cases of null and undefined

null:
Intentional Absence of Value: null is often used to indicate the intentional absence of any object value or a value that is explicitly set to nothing. It represents the idea that there is no value or object assigned to a variable or property.
Use null when you want to explicitly indicate the absence of a value or when you want to release references to objects.

undefined:
Default Initial Value: Variables declared but not initialized are assigned the value undefined by default. It represents the absence of any value assignment. This is typically the initial state of function parameters and variables.
Use undefined as the default value for uninitialized variables or function parameters and when you want to check for the absence of properties or function return values.

What do you mean by REST API?

REST API, which stands for Representational State Transfer Application Programming Interface, is a set of architectural principles and constraints for designing networked applications. It is an approach used in web services development to create scalable, stateless, and interoperable services that can be easily accessed over the internet. REST is not a specific technology but a set of conventions and best practices for building APIs.

REST APIs have become a widely adopted approach for building web services due to their simplicity, scalability, and compatibility with the HTTP protocol, making them suitable for a variety of applications and devices. They are commonly used in web and mobile app development to enable communication between clients and servers over the internet.
